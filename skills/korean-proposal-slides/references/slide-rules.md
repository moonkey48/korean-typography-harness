## 1. 공통 기하 — 1920×1080, 세이프 마진, 12컬럼

- **캔버스:** 1920×1080(16:9) 고정. 반응형 재배치 없음. 뷰포트에는 **통째 스케일**로 맞춘다.
- **세이프 마진:** 좌우 96px, 상하 64px. 이 안쪽이 콘텐츠 영역(1728×952px).
  - why: 프로젝터 오버스캔·TV 가장자리 잘림 방지. 텍스트가 화면 끝에 붙으면 뒷자리에서 압박감.
  - 96px는 슬라이드 전용 외곽 마진(48×2)으로 8그리드 위에 있다. 상하 64는 space 스케일 값.
- **12컬럼 · 거터 24px.** 콘텐츠 영역(1728px)을 12컬럼으로 나눈다.
  - 컬럼 폭 = (1728 − 24×11) / 12 = 122px. 본문은 7~9컬럼으로 묶어 행 길이를 통제.
  - 2단 구성: 좌 5~7컬럼 텍스트 + 우 5~7컬럼 차트/이미지.
- **푸터 영역:** 하단 세이프 마진 안에 페이지 번호·섹션·출처를 caption(14pt)으로.

## 2. pt→px 환산과 타입 스케일

발표 크기는 pt로 말하고, HTML 1920×1080 스테이지에서는 **1pt = 2px**로 렌더한다
(PowerPoint 16:9 = 13.333in×7.5in, 1080px/7.5in=144dpi → 1pt=2px, 동일 환산).

| 역할 | pt | px(1920) | line-height | tracking | 비고 |
|---|---|---|---|---|---|
| caption/출처 | 14 | 28 | 1.45 | 0 | 최소한만. 라이브면 지양 |
| body 본문 | **18** | 36 | 1.35~1.5 | 0 | 배포/인쇄 기준 |
| body-live | **24+** | 48+ | 1.4 | 0 | 라이브 발표 우선 |
| eyebrow/h4 | 18 | 36 | 1.4 | 0 | 라벨·꼬리표 |
| 부제/h3 | 24 | 48 | 1.4 | 0 | title 토큰 |
| 슬라이드 제목/h2 | 32 | 64 | 1.4 | 0 | headline 토큰 |
| 표지 제목/h1 | 44 | 88 | 1.3 | 0 | display 토큰 |

- 역할은 6~8개 이내로 고정. 미세하게 다른 크기 여러 개 금지(foundation 원칙 5).
- KPI 대형 수치는 h1(88px) 이상으로 키워도 되지만 tabular-nums 필수.

## 3. 슬라이드 종류별 규칙

### 표지 (cover)
- 슬롯: eyebrow(문서 종류/날짜) · h1 제목 · h3 부제 · 발표자/조직 caption.
- 제목만 예외적으로 중앙 또는 좌측 하단 정렬 허용. 나머지는 좌측.
- 배경은 surface.default 또는 muted. 이미지 배경이면 오버레이로 대비 3:1+ 확보.

### 본문 (body / 메시지)
- **한 슬라이드 = 한 메시지.** h2 제목 한 줄로 결론을 먼저.
- 본문은 BodyBlock(7~9컬럼, max-width로 한글 24~34자). 불릿은 3~5개, 각 1~2줄.
- "설명은 화면, 디테일은 발표 노트." 화면에 문장을 다 넣지 않는다.

### 차트 (chart) — ChartCard 6슬롯 필수
- `title(32pt) / subtitle(18pt) / chart / description / source(14pt) / download`.
- description·source·download는 **생략 불가**(배포·인쇄본 근거 추적). 발표 화면에선 source·download를
  하단 caption으로 작게 두되 슬롯은 유지.
- 차트 규칙(foundation): 시리즈 4개 이내, stacked 4범주 이내, clustered 4막대 이내,
  gridline 10 이하, 범례보다 직접 라벨링, 다중 시계열은 선 차트, 색 categorical 1~4 + focus.
- 차트 안 라벨은 18pt 이상(뒷자리 가독). 웹 차트 14px 라벨을 그대로 쓰지 않는다.

### 표 (table) — TableWrapper
- 숫자 우정렬·텍스트 좌정렬·셀 3줄 이내·세로 구분선 지양(행 구분 위주).
- 셀 패딩은 슬라이드에선 넉넉히(가로 16~24 / 세로 12~16px 상당), 행 높이 크게.
- 열·행이 많아 글자가 14pt 밑으로 내려가야 하면 → **표를 분할**하거나 요약표+상세 부록으로.

### KPI (지표) — KpiCard
- `eyebrow / metric / delta / help`. metric은 h1급 대형 수치.
- delta는 색+방향아이콘+수치(색 단독 금지). 상승/하락/보합 3상태.
- 한 슬라이드에 KPI 3~4개까지. 그 이상은 분할.

### 마무리 (closing)
- 핵심 메시지 재진술 1줄 + 다음 행동(CTA)/연락처. 장식 최소화.
- "감사합니다"만 있는 빈 슬라이드보다 요약·연락처가 유효.

## 4. "축소 금지 → 분할" 규칙 (제안서의 핵심)

스크롤이 없으므로 넘칠 때 **폰트·차트·표를 줄이면 안 된다.** 대신:

1. 메시지를 쪼갠다(한 슬라이드 = 한 메시지).
2. 차트가 크면 "요약 지표 슬라이드 → 상세 차트 슬라이드"로 나눈다.
3. 큰 표는 "요약표(핵심 행) + 부록 상세표"로 분리.
4. 불릿이 6개+면 두 슬라이드로.

기준: **본문이 18pt(라이브 24pt) 밑으로 내려가야 들어간다면, 그건 분할 신호다.**

## 5. 정렬 · 불릿 · 행 길이

- **왼쪽 정렬 기본.** 양끝(justify) 금지(한글 어절 간격 들쭉날쭉·구멍). 중앙정렬은 표지·짧은
  한 줄 강조만.
- **불릿 절제.** 슬라이드당 3~5개, 2단계 깊이까지. 문장형 불릿 대신 키워드+짧은 설명.
- **행 길이:** 한글 24~34자 / 혼합 45~65자. BodyBlock max-width로 강제. 1920 전폭에 본문을
  깔면 한 줄 60자+가 되어 뒷자리에서 눈이 튕긴다 → 7~9컬럼으로 묶는다.

## 6. 색 · 대비 (투사 보정)

- foundation semantic 색 그대로: text.primary #111 / secondary #4B5563, surface.default #FFF /
  muted #F7F8FA. **회색 본문(text.tertiary #6B7280)은 본문에 쓰지 않는다.**
- 대비: 본문 4.5:1, 큰 텍스트 3:1. 투사는 명암 손실이 크므로 **여유 있게**(밝은 회의실 대비).
- 차트 색: categorical #2563EB / #059669 / #D97706 / #7C3AED, focus #2563EB, muted #9CA3AF.
  인접 색 대비 3:1, 색 단독으로 의미 전달 금지(라벨 병기).
- 어두운 배경 슬라이드를 쓸 땐 semantic을 뒤집어 설계(순흑·순백 지양, 대비 유지).

## 7. 흔한 실패 카탈로그 (변환 시 우선 점검)

| 증상 | 왜 나쁜가 | 고치기 |
|---|---|---|
| 14~16pt 본문 | 뒷자리에서 안 읽힘 | 18pt(라이브 24pt+)로 상향 |
| 라틴용 과대 tracking | 한글이 성기고 어수선 | tracking 0 리셋 |
| 중앙·양끝 정렬 남발 | 어절 간격 붕괴·읽기 리듬 깨짐 | 왼쪽 정렬 |
| 불릿 8~10개 | 한 슬라이드 다중 메시지 | 3~5개로 압축·분할 |
| 차트/표 축소해 끼워넣기 | 투사에서 라벨 죽음 | 원 크기 복원 후 분할 |
| 슬라이드당 차트 3개+ | 메시지 분산 | 1~2개로, 나머지 분할 |
| source·description 없는 차트 | 근거 추적 불가(공공/IR) | ChartCard 6슬롯 채움 |
| 텍스트 화면 끝에 붙음 | 오버스캔 잘림·압박 | 96/64 세이프 마진 |
| 회색 본문(#6B7280) | 대비 미달·촌스러움 | text.primary/secondary |
| 임의 여백(37px 등) | 리듬 붕괴 | 4/8 스케일·거터24 |

## 8. 검수 포인트 (모드 C 체크리스트)

각 항목 **PASS / FAIL / N/A**. FAIL은 슬라이드 번호·실제값·수정안 기록.

- [ ] 본문 18pt 이상(라이브 24pt+), caption 14pt 미만 없음
- [ ] 줄간격 1.35~1.5(1.35 미만·1.6 초과 없음)
- [ ] 본문·라벨 자간 0, 제목 비영 자간은 검수 근거 있음
- [ ] 왼쪽 정렬(중앙·양끝 남용 없음)
- [ ] 세이프 마진 좌우96/상하64 준수, 가장자리 붙음 없음
- [ ] 12컬럼·거터24, 4/8 스케일 밖 임의 여백 없음
- [ ] 한 슬라이드 = 한 메시지, 불릿 6개 미만
- [ ] 행 길이 한글 24~34자/혼합 45~65자(BodyBlock max-width)
- [ ] 차트/표 축소 대신 분할했는가
- [ ] ChartCard 6슬롯(특히 source·description) 충족
- [ ] 표 숫자 우정렬·텍스트 좌정렬·셀 3줄 이내·세로선 지양
- [ ] KPI delta 색+아이콘+수치(색 단독 아님), metric tabular-nums
- [ ] 본문 대비 4.5:1·큰 텍스트 3:1, 회색 본문 없음
- [ ] 차트 시리즈 4 이내·gridline 10 이하·색 categorical 1~4
- [ ] 200% 확대/화면공유 축소에도 내용 손실 없음
