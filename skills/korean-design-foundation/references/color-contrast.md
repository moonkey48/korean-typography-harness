# 색 · 대비

## 색 시맨틱 (기본, light)

| 토큰 | 값 | 용도 |
|---|---|---|
| text.primary | `#111111` | 본문·제목 기본 |
| text.secondary | `#4B5563` | 보조 텍스트 |
| text.tertiary | `#6B7280` | 캡션·비활성 |
| surface.default | `#FFFFFF` | 기본 배경 |
| surface.muted | `#F7F8FA` | 카드·섹션 배경 |
| surface.elevated | `#FFFFFF` (+shadow) | 떠 있는 표면 |

원칙: **색보다 구조 먼저.** 색은 정보 구조의 마지막 보정 장치다. 과한 accent·높은 채도·낮은 대비 회색 본문은 한국어 화면을 가장 쉽게 촌스럽게 만든다.

## WCAG 대비 (필수)

| 대상 | 최소 대비 |
|---|---|
| 일반 텍스트 | **4.5:1** |
| 큰 텍스트(24px+ 또는 19px+ bold) | **3:1** |
| 비텍스트 그래픽(아이콘·차트 요소·경계) | **3:1** |

- `text.tertiary`(#6B7280)를 본문에 쓰지 않는다 — 회색 본문 남용은 대비 미달·촌스러움의 주범.
- 이미지 위 카피는 오버레이/그라디언트로 대비 확보.
- 버튼의 모든 상태(default/hover/focus/active/disabled) 대비를 초기에 검수.

## 차트 색 원칙

- categorical 팔레트는 **5색 미만**으로 제한.
- 인접 색 대비 **3:1** 확보 (색맹 고려, 색만으로 구분하지 않기).
- 순차형(sequential)은 단일 hue 명도 스케일.
- focus 색 1개로 "강조 시리즈" 지정, 나머지는 저채도.
- 가능하면 범례보다 직접 라벨링 → 색 의존도를 낮춘다.

## 시맨틱 상태색 (권장)

- success / warning / danger / info는 각각 텍스트 대비 4.5:1을 만족하는 명도로.
- 상태를 **색 단독**으로 전달하지 않는다 (아이콘·텍스트 병기).

## 다크모드 방향

- 이 SSOT 기본은 light. dark는 semantic 토큰을 뒤집는 구조로 확장:
  - text.primary → 밝은 회색(`#F2F2F2`대), surface.default → 어두운 배경(`#111`대).
  - 순수 흑(#000) 배경·순수 백(#FFF) 텍스트는 피하고 대비는 유지.
- 다크는 처음부터 semantic 토큰으로 설계 (하드코딩 hex 금지)해야 전환이 쉽다.
