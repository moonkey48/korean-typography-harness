# 마스터 검수 체크리스트

운영에서 중요한 것은 "완성 느낌"이 아니라 **검수 가능성**이다. 각 항목은 **숫자 기준 + 검사 도구 + 통과/실패**로 판정한다. 디자이너·개발자 공용.

| 검수 항목 | 통과 기준 | 검사 도구 |
|---|---|---|
| 본문 크기 | 웹/앱 본문 **16px·16sp·16pt 이상**, 예외는 dense table만 15 | 디자인 리뷰 + ESLint/stylelint |
| 줄간격 | 본문 **1.5 이상**, 긴 문단 1.6 내외 | Figma style audit + CSS lint |
| 자간 | 한글 본문/라벨 **0em**, 제목도 기본 **0em**, 비영 자간은 검수 근거 필요 | 디자인 토큰 검사 |
| 제목 대비 | 제목/본문 비율 **1.25~1.5배** | typography token diff |
| 마진/거터 | 토큰 값만 사용, small 16, medium+ 24 | stylelint custom plugin |
| 색 대비 | 일반 **4.5:1**, 큰 텍스트 **3:1**, 비텍스트 **3:1** | Figma checker, Stark, axe |
| 확대 대응 | 200% 손실 없음, 대시보드 400%도 확인 | 브라우저 zoom, Playwright |
| 터치 목표 | iOS **44pt**, Android **48dp** 이상 | 시뮬레이터/Inspector |
| 차트 시리즈 | 일반 **4 이내**, stacked **4범주 이내**, gridline **10 이하** | 수동 리뷰 + 차트 템플릿 |
| 표 규칙 | 숫자 우정렬, 텍스트 좌정렬, 셀 3줄 이내, vertical line 지양 | 컴포넌트 QA |
| 대시보드 구조 | 중요 지표 상단, horizontal scroll 지양, 큰 표 pagination | UX/QA 리뷰 |
| 코드 일관성 | raw px/hex 직접 사용 최소화, token 참조 강제 | ESLint/stylelint + PR review |

## 사용성 (P0-7)

정본(SSOT): `usability-heuristics.md`(P0-1) — 각 항목의 근거·합/불 예시·휴리스틱 매핑은 그 문서를 따른다. 아래는 게이트 판정용 요약이다. 판정 단계는 `korean-design-checks` 파이프라인 기준(1 디자인 / 2 핸드오프 / 3 코드 / 4 렌더 / 5 카피)이며, 판정 기록은 아래 "판정 규칙"(PASS/FAIL/N/A)을 동일하게 따른다.

| 검수 항목 | 통과 기준 | 검사 도구 | 판정 단계 | 자동화 |
|---|---|---|---|---|
| async 액션 피드백 | 모든 async 액션(저장·제출·삭제)에 로딩·성공·실패 피드백 존재 | Playwright | 4 렌더 | 자동 |
| 인라인 검증 타이밍 | 폼 인라인 검증은 **blur + submit** 타이밍(입력 중 실시간 에러 지양) | Playwright | 4 렌더 | 자동 |
| 에러 카피·입력 보존 | 에러 메시지는 **원인+해결** 안내, 실패 시 사용자 입력 보존 | Playwright(입력 보존) + 카피 리뷰 | 4 렌더 + 5 카피 | 자동(입력 보존) / 수동(카피) |
| focus-to-first-error | 제출 실패 시 첫 에러 필드로 포커스 이동, 에러가 필드에 연결(`aria-describedby`) | Playwright + axe | 4 렌더 | 자동 |
| 파괴적 액션 확인/undo | 삭제 등 파괴적 액션에 확인 단계 **또는** undo 제공 | Playwright | 4 렌더 | 자동 |
| 목록·표 3상태 | 목록·표마다 로딩·빈·에러 3상태 구현 | 정적 리뷰 + Playwright(네트워크 모킹) | 3 코드 + 4 렌더 | 자동(모킹 가능 시) / 불가 시 수동 |
| 모달·멀티스텝 탈출 경로 | 모달 ESC/닫기, 멀티스텝 뒤로가기 존재 — 막다른 길 금지 | Playwright | 4 렌더 | 자동 |
| 내비 현재 위치 | 현재 위치 표시(활성 내비 상태·브레드크럼 — "내가 어디 있는지") | UX 리뷰 | 1 디자인 + 4 렌더 | 수동 |

## PR 템플릿에 넣을 항목

- 차트 대체 설명(alt/source/download) 여부
- table alignment 여부
- 200% 확대 체크 여부

- 한글 자간 오버튜닝 여부
- 표·차트 카드 슬롯 충족 여부 (ChartCard 6슬롯 / TableWrapper)

## 판정 규칙
- 각 항목은 **PASS / FAIL / N/A**로 기록.
- FAIL은 근거 수치와 함께 기록하고 수정 제안을 붙인다.
- 상충 데이터는 삭제하지 않고 출처 병기.
