# 간격 · 그리드 · 반응형

## Space scale (4/8 시스템)

**4px 서브그리드 + 8px 매크로 스케일.** 간격은 아래에서만 고른다:

`4 · 8 · 12 · 16 · 24 · 32 · 40 · 48 · 64` (px)

- 4px: 미세 조정(아이콘-텍스트 간격 등)
- 8px 배수: 기본 리듬
- 본문 블록·카드·패널·섹션 간격은 이 계열에서만 선택
- **임의 여백 금지.** 어색함의 대부분은 서체가 아니라 spacing에서 온다.

### 시맨틱 간격 (권장 매핑)
- `space.inline` = 8~12px (인라인 요소 간)
- `space.card` = 16~24px (카드 내부 패딩·카드 간격)
- `space.section` = 40~64px (섹션 간)
- 패딩/마진 네이밍은 **-x / -y** 접미사 (KRDS 권장): `padding-x`, `padding-y`.

## Breakpoints

| 이름 | 값 | 대상 |
|---|---|---|
| sm | 360px | 모바일 |
| md | 768px | 태블릿 |
| lg | 1024px | 소형 데스크톱 |
| xl | 1280px | 데스크톱 |
| xxl | 1440px | 대형 |

모바일 앱(창 기준): Compact <600dp / Medium 600–839dp / Expanded 840dp+.

## 그리드 (KRDS)

| 구간 | 컬럼 |
|---|---|
| small | 4–6 |
| medium | 8–12 |
| large | 12–16 |

- **마진:** small 16px, medium 이상 24px.
- **거터:** 최소 16px, 권장 24px.
- 유형별 컬럼: 랜딩 모바일 4 / tablet 8 / desktop 12. 대시보드 desktop 12 / tablet 8 / mobile 4.

## 반응형 전환 규칙

- **차트:** 모바일 1열 → tablet 2열 → desktop 3~4열.
- **표:** 좁으면 **카드형 전환** 또는 **가로 스크롤 + 헤더 고정**. 큰 표는 pagination.
- **대시보드:** 모바일 세로 1열 → tablet 2열 → desktop 3~4열. horizontal scroll 회피.
- **제안서:** 반응형이 아니라 **투사 거리** 기준 (1920×1080 고정, 좌우 96px·상하 64px).

## 카드·컨테이너 기본 여백

- 차트 카드 패딩: 24px(데스크톱) / 16px(모바일)
- 표 셀 패딩: 12×16px, 행 높이 44~48px
- 카드 간격(대시보드): 16~24px
- 본문 블록: **max-width 필수** (한글 24~34자 / 혼합 45~65자, 예 `32rem`)
