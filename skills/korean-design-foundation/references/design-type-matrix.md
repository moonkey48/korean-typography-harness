# 디자인 유형별 수치 매트릭스

> KRDS 본문·계층·레이아웃, WCAG 대비, Material 8dp·적응형 breakpoint, Apple/Android 모바일 규칙, Microsoft 제안서 접근성을 종합한 **운영 권장값**. 직접 규격이 아니라 한국어 맥락 즉시 적용 표준안. 원칙: 본문은 라틴 대비 1단계 크게, tracking은 더 보수적으로, 여백은 더 규칙적으로.

## 2. 차트 예외 규칙

- **시리즈 수: 일반 4개 이내.**
- **stacked bar: 스택당 4범주 이내.**
- **clustered bar: 한 클러스터 4개 막대 이내.**
- **gridline: 최대 10개 전후.**
- 가능하면 **범례보다 직접 라벨링**을 쓴다.
- 다중 시계열을 바 차트로 표현하지 않는다 (선 차트 사용).
- 색은 5개 미만으로 제한하고, 인접 색 대비 3:1 확보.
- 차트에는 **대체 설명(alt) · 데이터 소스 · 데이터 다운로드**를 제공한다.

## 3. 표(table) 규칙

- **숫자 데이터: 오른쪽 정렬.** 텍스트: 왼쪽 정렬.
- **셀 내 텍스트 3줄 이내.**
- **vertical line(세로 구분선) 지양** — 행 구분 위주.
- 셀 패딩 12×16px, 행 높이 44~48px.
- 좁은 화면: **카드형 전환** 또는 **가로 스크롤 + 헤더 고정**.
- 큰 표: **pagination** 사용. 모달 안에 큰 표 삽입 금지.

## 4. 대시보드 구조 규칙

- **inverted pyramid**: 중요 지표를 상단에 먼저.
- 최소 스크롤, **horizontal scrolling 회피**.
- 대형 표는 pagination.
- 한 페이지에 chart/table 과적재 금지, 내부 스크롤 지양.
- 대시보드 텍스트 최소 12pt 이상(하한). 실무는 본문 16 기준으로 시작.

## 5. 터치·확대 규칙

- 터치 목표: **Android 48dp × 48dp, iOS 44pt × 44pt** 이상. 한국어 UI에서 이 하한을 밑돌지 않는다.
- 확대: **200% 확대 시 내용/기능 손실 없음.** 대시보드·웹 콘텐츠는 **400%도 확인.**
- 모바일은 Dynamic Type / fontScale을 무시하지 않는다.

## 6. 행 길이 규칙

- Baymard 최적 라틴 행 길이 50~75자. 한국어는 블록 밀도가 높아 체감이 빨리 길어짐.
- **한글 문장 24~34자**, 또는 **혼합 텍스트 45~65자**로 짧게 잡는다.
- `BodyBlock`·긴 본문형 콘텐츠는 반드시 **max-width**를 둔다 (예: `max-width: 32rem`). 없으면 화면이 넓어질수록 행이 길어져 가독성이 무너진다.
