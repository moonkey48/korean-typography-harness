# Hero Visual Art Direction — 히어로 비주얼 아트디렉션

랜딩·웹앱 첫 화면의 **비주얼 히어로**를 아트디렉팅한다. "비주얼을 넣을지 말지"부터 시작해, 넣기로 했다면 구도 → 헤드라인 통합 → 그레이딩 → 반응형까지 한 번에 결정한다. 결과물은 하네스 **G2 디자인 방향 게이트**의 direction doc에 편입되는 **비주얼 방향 보드**다.

수치와 근거는 지어내지 않고 아래를 따른다:

- `../hallmark/references/hero-enrichment.md` — 이미지 필요성 게이트, E1~E8 아키타입, HP1~HP4 폴리시, 8문항 품질바 *(optional companion — hallmark 스킬 설치 시 형제 경로로 해석, 미설치 시 이 스킬 내부 규칙으로 폴백)*
- `../hallmark/references/imagery-kit.md` — 레이어드 히어로 합성 패턴, 블렌드 앤티패턴 *(optional companion — hallmark)*
- `../korean-design-foundation/references/color-contrast.md` — 오버레이 대비 기준 (일반 4.5:1 / 큰 텍스트 3:1)
- `../korean-design-foundation/references/type-scale.md` — display 스케일(44px·lh 1.3·자간 0em), 짧은 대형 카피 원칙
- `../korean-design-foundation/SKILL.md` — 행 길이(한글 24~34자 / 혼합 45~65자)
- `../korean-web-layout/references/landing.md` — Hero 헤드라인 44~60px, 이미지 위 저대비 카피 금지, center 남용 금지
- `../brand-image-pipeline/SKILL.md` — 온브랜드 이미지 생성·후처리 파이프라인 (이 스킬의 소스 공급자) *(optional companion — 이 번들에 미동봉)*

## ① 비주얼 히어로 vs 타이포-온리 — 선택 기준표

**hallmark의 디폴트는 타이포-온리다(hero-enrichment Tier 0: "Better nothing than bad something").** 이 스킬은 그 디폴트를 뒤집지 않는다 — 언제 디폴트를 *떠나도 되는지*를 판정한다. "첫 화면이 밋밋해" 요청이 곧 "이미지를 넣어라"는 뜻이 아니다. 밋밋함의 1차 처방은 hallmark HP1~HP4 폴리시 패턴(수직 레일·마퀴 오버플로·스포트라이트·장식 숫자)이고, 비주얼은 아래 3축 판정을 통과할 때만 꺼낸다.

| 판단 축 | 타이포-온리 유지 (디폴트) | 비주얼 히어로로 전환 |
|---|---|---|
| **브랜드 자산 유무** | 실사진·렌더·푸티지·일러스트 자산이 없다 (placeholder뿐이다) | 자기 소유 사진/제품 렌더/실제 화면 녹화가 있다, 또는 `brand-image-pipeline`으로 온브랜드 생성이 가능하다 |
| **제품 실체** | 개념·API·라이브러리·문서 — 보여줄 물체가 없다 | 물리적 제품·공간·음식·사람·실제 UI 화면 — 보여줄 실체가 있다 |
| **감성 소구 여부** | 정보·신뢰·효율 소구 (B2B SaaS, dev-tool, 스펙 중심) | 분위기·욕구·라이프스타일 소구 (커머스·F&B·여행·뷰티·브랜드 캠페인) |

판정 규칙:

1. **3축 중 2축 이상**이 오른쪽 열이어야 비주얼로 전환한다. 1축뿐이면 타이포-온리 + HP 폴리시로 해결한다.
2. 전환하더라도 hero-enrichment의 **8문항 품질바**(전달력·2MB·reduced-motion·삭제 생존 테스트 등)를 전부 통과해야 출하한다. 하나라도 no면 타이포-온리로 되돌린다.
3. **삭제 테스트가 최종심이다.** 비주얼을 지워도 히어로(헤드라인+lede+CTA)가 성립해야 한다. 비주얼이 무너진 타이포를 떠받치고 있다면 그 비주얼은 자격이 없다.
4. 자산이 없는데 감성 소구가 필요하면 스톡을 깔지 말고 `brand-image-pipeline`으로 온브랜드 생성부터 한다 — 그것도 불가하면 타이포-온리다.

## ② 구도 체계 — 초점·레이어·네거티브 스페이스·시선

### 1초점 원칙

히어로의 시각적 초점(얼굴·제품·핵심 모티프)은 **정확히 1개**다. 초점이 2개면 시선이 분산되고, 0개(균질한 텍스처·패턴뿐)면 비주얼이 벽지로 전락해 ①의 삭제 테스트를 통과하지 못한다. 초점 후보가 여럿이면 크롭으로 하나만 남긴다.

### 레이어 깊이 3층

| 층 | 역할 | 규율 |
|---|---|---|
| **배경층** | 그레이딩된 이미지·워시·그라디언트 — "종이" 역할 | 저채도·저대비. 헤드라인과 경쟁 금지. 페이지 전역이 아닌 히어로 스코프 |
| **주제층** | 초점 1개 (제품·인물·목업·일러스트) | 한쪽으로 치우쳐 배치(imagery-kit: 중앙 배치는 AI 디폴트). 생각보다 크게 |
| **전경 액센트** | 작은 타이포 라벨·오너먼트·그레인 중 **최대 1개** | 없어도 된다. 2개 이상이면 소음 |

3층이 전부 있어야 하는 게 아니다 — 배경+주제 2층이 기본이고, 전경 액센트는 페이지가 밋밋할 때만 1개 추가한다.

### 네거티브 스페이스 — 헤드라인 존 먼저 예약

구도를 잡는 순서는 이미지가 아니라 **헤드라인 존이 먼저**다.

- 히어로 폭의 **40% 이상**을 헤드라인 존(헤드라인+lede+CTA)으로 먼저 예약하고, 이미지의 빈 영역(하늘·벽·아웃포커스·단색면)이 그 존에 오도록 크롭·배치한다.
- **초점과 헤드라인 존이 겹치면 구도 실패다.** 스크림을 더 진하게 까는 것은 치료가 아니라 증상 은폐 — 크롭을 다시 잡는다.
- 생성 이미지를 쓸 때는 `brand-image-pipeline` 프롬프트에 네거티브 스페이스 방향("좌측 1/2 여백", "상단 여백")을 명시해 텍스트 존을 이미지 단계에서 확보한다.

### Z/F 시선 패턴과 한글 좌상단 관례

한글도 좌→우·상→하로 읽는다. 히어로의 기본 관례:

- **헤드라인은 좌상단 시작**이 기본값. 시선의 출발점을 고정한다.
- 초점(주제층)은 **우측 또는 우하단**에 배치해 좌상(헤드라인)→우하(초점/CTA)의 **Z 대각선**을 만든다. 텍스트가 많은 히어로(스펙·비교 중심)는 F 패턴 — 좌측 정렬 블록을 세로로 쌓는다.
- 가운데 정렬은 헤드라인 한두 줄까지만 허용(landing.md: 과도한 center는 한글 긴 문장에서 시선 시작점이 흔들린다). eyebrow·lede·CTA까지 전부 중앙 스택은 AI 지문이다(hero-enrichment 공간 규율과 동일).

## ③ 이미지 ↔ 한글 헤드라인 통합

### 오버레이 대비 — color-contrast.md 기준

이미지 위 카피의 대비는 `korean-design-foundation/references/color-contrast.md`를 따른다: **일반 텍스트 4.5:1, 큰 텍스트(24px+ 또는 19px bold+) 3:1**. 히어로 헤드라인은 display급(44px+)이라 3:1이 하한이지만, lede·eyebrow는 4.5:1이다. "밝은 배경에 흰 글씨 금지"(landing.md). 대비는 이미지의 **가장 밝은/어두운 지점**에서 측정한다 — 평균이 아니라 최악 지점이 기준이다.

### 스크림/그라디언트 오버레이 레시피 3종

헤드라인 존 위치에 맞춰 하나만 고른다. 전면 균일 스크림은 이미지를 죽이는 마지막 수단이다.

```css
/* A. 방향성 그라디언트 스크림 — 헤드라인 존 쪽만 어둡게 (기본 권장) */
.hero__scrim {
  position: absolute; inset: 0; z-index: 1; pointer-events: none;
  background: linear-gradient(to right,
    color-mix(in oklch, var(--color-ink) 65%, transparent) 0%,
    color-mix(in oklch, var(--color-ink) 35%, transparent) 45%,
    transparent 70%);
}

/* B. 하단 고정 스크림 — 풀블리드 사진 + 하단 헤드라인 존 */
.hero__scrim--bottom {
  background: linear-gradient(to top,
    color-mix(in oklch, var(--color-ink) 72%, transparent) 0%,
    transparent 55%);
}

/* C. 로컬 스크림 — 초점을 살려야 해서 전면 스크림이 아까울 때 */
.hero__copy-panel {
  background: color-mix(in oklch, var(--color-paper) 78%, transparent);
  backdrop-filter: blur(12px);
  padding: var(--space-xl);
}
```

텍스트는 `z-index: 2`로 스크림 위에 놓는다. 스크림 농도는 "대비 계산이 통과하는 최소값"으로 잡는다 — 습관적 `rgba(0,0,0,.6)` 전면 도포는 그레이딩(④)을 무의미하게 만든다.

### 한글 줄바꿈 — word-break: keep-all

이미지 위 헤드라인은 행폭이 좁아 어절 중간 꺾임이 특히 잘 일어난다. 헤드라인·lede·버튼 라벨에 필수:

```css
.hero__display, .hero__lede, .hero .btn {
  word-break: keep-all;       /* 어절 단위 줄바꿈 */
  overflow-wrap: break-word;  /* 긴 URL·영문 오버플로 방지 */
}
.hero__display { text-wrap: balance; } /* 2줄 헤드라인 좌우 균형 */
```

### 헤드라인 행길이·스케일 — type-scale.md 기준

- 스케일: `type-scale.md` — display **44px**(웹 히어로 44~60px 범위, landing.md), line-height **1.3**(대형 카피는 1.2~1.35로 좁혀 덩어리감), 자간 **0em**(비영 자간은 렌더링 검수 근거가 있을 때만).
- 행길이: foundation의 행 길이 규칙(본문 한글 24~34자)은 **본문 기준**이다. display 헤드라인은 "짧은 대형 카피" — **한 줄 8~16자, 최대 2줄**(관례 — foundation 수치 아님, 랜딩 display 카피 통용 범위)로 그보다 훨씬 짧게 잡고, 헤드라인 블록에 `max-width`를 걸어 이미지 초점을 침범하지 않게 한다. lede는 BodyBlock 규격(한글 24~34자 ≈ 32rem)을 그대로 쓴다.

## ④ 컬러 그레이딩 — 이미지를 브랜드 팔레트에 종속시키기

원본 사진의 색이 페이지 팔레트와 따로 노는 것이 "갖다 붙인 스톡" 인상의 주범이다. 이미지를 페이지에 종속시키는 CSS 절차 (위에서부터 순서대로, 필요한 단계까지만):

1. **채도 정리** — 원본 채도를 페이지 톤 아래로 끌어내린다. `filter: saturate(0.75) contrast(1.02);` (hallmark E8 예시의 "8% 탈채도"와 같은 원리 — 사진이 페이지의 종이 톤과 화해하게 한다.)
2. **톤 통일(틴트)** — 브랜드 anchor hue 쪽으로 전체를 기울인다. 이미지 위에 브랜드색 레이어 + 블렌드:
   ```css
   .hero__media { position: relative; isolation: isolate; }
   .hero__media::after {
     content: ""; position: absolute; inset: 0; pointer-events: none;
     background: color-mix(in oklch, var(--color-accent) 18%, transparent);
     mix-blend-mode: color;   /* 색상만 주입, 명암 보존 */
   }
   ```
3. **듀오톤(강한 종속)** — 브랜드 2색으로 완전히 재채색. 배경/분위기용 이미지에만 쓰고, 제품 실물색이 중요한 커머스에는 금지:
   ```css
   .hero__media img { filter: grayscale(1) contrast(1.05); mix-blend-mode: luminosity; }
   .hero__media { background: linear-gradient(135deg, var(--color-ink), var(--color-accent)); }
   ```
4. **그레인 마감** — 합성 티를 지우는 최종 텍스처. `<feTurbulence>` 오버레이 ≤ 0.08 opacity (hero-enrichment E7 레시피). 테마가 이미 종이 질감이면 생략(그레인 중복은 탁해진다).

규율:

- 블렌드는 반드시 **활성 배경색 위에서 눈으로 검증**한다 — `multiply`는 다크 배경에서 뒤집히고 `screen`은 라이트에서 날아간다(imagery-kit 앤티패턴).
- 그레이딩 후 **③의 대비를 재측정**한다. 틴트·듀오톤은 텍스트 대비를 쉽게 무너뜨린다.
- CSS 그레이딩은 "온브랜드에 가까운 소스"를 마감하는 도구다. 소스 자체가 오프브랜드(스톡 티, 엉뚱한 조명·색)라면 CSS로 구제하지 말고 **`brand-image-pipeline` ④ 후처리 단계**(그레인·비대칭 크롭·컬러 그레이드·듀오톤을 이미지 파일 레벨에서 수행)로 되돌려 소스를 다시 만든다. 파이프라인이 파일에 굽고, 이 스킬은 CSS로 페이지 톤에 최종 정합시킨다 — 역할 분담이다.

## ⑤ 히어로 아키타입 — hallmark E1~E8 + 풀 비주얼 변형

hallmark hero-enrichment의 8개 아키타입을 기저로 쓰되, ①에서 "비주얼 히어로"로 판정된 브리프를 위해 각각에 **풀 비주얼 변형**(비주얼이 폴드의 주인공이 되는 확장)을 1줄씩 추가한다. 원형의 use-when/avoid-when·코드 스케치는 hero-enrichment를 그대로 따른다.

| # | 원형 (hallmark) | 풀 비주얼 변형 |
|---|---|---|
| E1 | 데모 비디오 — 뷰포트 우측 클립 | 클립을 유지한 채 비디오를 폴드 높이 전체로 키우고, 헤드라인은 좌측 방향성 스크림(③-A) 존에 오버레이한다 |
| E2 | 풀블리드 뮤트 루프 + 고스트 오버레이 | 고스트 틴트를 걷어 푸티지 자체를 주제로 승격하고, 하단 스크림(③-B) 40%에 헤드라인 존을 고정한다 |
| E3 | 브라우저 프레임 목업 스플릿 | 같은 목업을 확대·블러·그레이딩해 배경층으로 깔고, 선명한 프레임 목업 1개를 주제층으로 겹쳐 3층 깊이를 만든다 |
| E4 | 프레임 없는 플로팅 스크린샷 | 스크린샷을 퍼스펙티브 틸트(2~4°)로 폴드 우측 55~60%를 채우는 히어로 오브젝트로 확대하고 우측 엣지에서 클립한다 |
| E5 | 커스텀 일러스트 센터피스 | 일러스트를 풀블리드 씬으로 확장하되, 제작 단계에서 헤드라인 존(폭 40%+)을 네거티브 스페이스로 미리 비워 그린다 |
| E6 | 소형 CSS/SVG 애니메이션 루프 | 루프 모티프를 배경층 전면으로 확장한다 — 단 브랜드 모티프 기반이어야 하며, 범용 파티클·오로라로 흐르면 즉시 타이포-온리로 회귀 |
| E7 | 추상 배경 — 저채도 그라디언트+그레인 | 그라디언트를 브랜드 듀오톤(④-3) 이미지 워시로 승격한다 — 이미지가 생겨도 여전히 "종이" 역할이며 초점은 헤드라인에 남는다 |
| E8 | 단일 타이트 크롭 사진 | 사진을 풀블리드로 확장하고 초점을 3분할 우측 교점에 재크롭, 좌상단 빈 영역을 헤드라인 존으로 확보한다(②의 Z 대각선) |

규율: 풀 비주얼 변형은 원형보다 **실패 비용이 크다**. 8문항 품질바(특히 2MB 예산·LCP·reduced-motion)와 ③의 대비 기준을 원형보다 엄격하게 적용하고, 확신이 없으면 원형으로 한 단계 내려온다. 폴리시 패턴(HP1~HP4)은 풀 비주얼 위에 겹치지 않는다 — 비주얼이 이미 구조적 선택이다.

## ⑥ 반응형 — 모바일에서 크롭 vs 제거

데스크톱 히어로를 모바일(360~430px)로 내릴 때, 비주얼을 **세로 크롭으로 유지**할지 **제거하고 타이포-온리로 폴백**할지를 아래 기준으로 결정한다. "축소해서 우겨넣기"는 선택지가 아니다.

| 판정 질문 | 예 → 크롭 유지 | 아니오 → 제거(타이포-온리 폴백) |
|---|---|---|
| 초점이 세로 크롭(3:4·9:16)에서 살아남는가? | 인물·제품 단일 초점 | 가로로 긴 구도 — 대시보드 목업·클립드 비디오·와이드 씬 |
| 모바일 이미지 예산(≤200KB, LCP 2.5s)을 지킬 수 있는가? | WebP/AVIF 별도 소스 준비 가능 | 비디오·대형 씬으로 예산 초과 |
| 크롭 후에도 헤드라인 존 + 대비 4.5:1이 확보되는가? | 빈 영역이 세로 크롭에 남는다 | 초점과 텍스트가 겹쳐 스크림 떡칠이 필요해진다 |

- 3문항 **전부 예**일 때만 크롭 유지. 구현은 `<picture>` art direction — 모바일 전용 세로 크롭 소스를 별도 제공하고 `object-position`으로 초점을 고정한다. CSS `object-fit: cover` 하나로 데스크톱 원본을 재사용하는 것은 초점 유실의 지름길이다.
- 하나라도 아니오면 제거한다. hallmark E1 규칙과 동일 — 클립드 비디오는 60rem 미만에서 스택으로 접고 클립을 버린다("모바일 클립은 깨진 것처럼 보인다"). 제거 폴백은 ①의 삭제 테스트를 통과한 히어로라면 아무 문제 없이 성립한다 — 통과 못 했다면 애초에 출하 자격이 없던 것이다.
- 제거 시 배경층(E7급 워시·그라디언트)만 남기는 절충도 허용된다 — 전송량이 거의 없고 헤드라인 존을 침범하지 않는 경우에 한한다.

### 하네스 게이트 연결 (G2)

이 스킬은 한글 디자인 하네스 4게이트 파이프라인(리서치→방향→셀프리뷰→최종QA)의 **G2 디자인 방향 게이트**에서 호출된다 — 비주얼 배선 단계(`brand-image-pipeline`가 이미지 소스를 공급하고 `hero-visual-art-direction`이 배치·그레이딩을 결정).

- **입력**: G1 근거 브리프(레퍼런스 ≥3·타깃·톤). 브리프 없이 이 스킬로 히어로 방향을 확정하지 않는다.
- **산출물**: **비주얼 방향 보드** — ① 판정 결과(타이포-온리/비주얼 + 3축 근거), 선택 아키타입(E1~E8 또는 풀 비주얼 변형)과 구도 스케치(초점·헤드라인 존), 스크림·그레이딩 레시피, 모바일 크롭/제거 전략. 이것이 G2 direction doc의 "이미지 사용 시 비주얼 방향 보드" 슬롯을 채운다.
- **통과 기준**: 방향 보드 없이 히어로 구현에 진입하지 않는다. 구현 후 검증은 G3 셀프리뷰 루프(playwright 스크린샷 데스크톱+모바일 → design-review-mcp)와 G4 최종 QA(korean-design-checks — keep-all·대비 검사 포함)가 담당한다.
