# 랜딩 신뢰·전환 관례 (한글 B2B)

> 랜딩의 **신뢰 요소·CTA·리드폼·요금제** — "무엇을 어디에 놓아 신뢰를 만들고 전환시키는지"를 정한다. 타이포·그리드·간격 수치는 `landing.md` 소관. 각 규칙에는 출처가 있으면 출처를, 국내 B2B 랜딩의 통용 패턴이면 '관례'를 적는다.

## 1. 신뢰 요소 5종

| 요소 | 규칙 | 근거 |
|---|---|---|
| 고객사 로고 월 | 6~12개, 1~2행 고정. **그레이스케일 단색 통일**(hover 시 원색 복원은 선택). 배치는 Hero 바로 아래. | 관례 — 원색 로고 나열은 팔레트를 깨고, 3개 이하는 빈약해 보인다 |
| 도입 사례/후기 | **실명+직함+회사명** 3요소 필수, 얼굴 사진과 수치("응답 시간 40% 단축")를 붙일수록 강하다. 익명 후기("A사 담당자")는 수 채우기로 쓰지 않는다. | 관례 — 검증 가능성이 없는 후기는 신뢰 기여가 없다 |
| 인증마크 | ISO 27001·ISMS(-P)·GS인증·CSAP 등. **푸터 또는 요금표·리드폼 인접**에 소형 배치. Hero에 크게 넣지 않는다. | 관례 — 인증은 의심 해소 장치라 정보 입력·결제 판단 직전에 효과가 난다 |
| 언론 보도 | 매체 로고 + 기사 제목 + 날짜, 원문 링크. 로고만 나열하면 고객사 로고 월과 구분이 안 된다. | 관례 |
| 성과 숫자 | "도입 기업 1,200+", "누적 처리 3.2억 건" — 검증 가능한 수치만. tabular-nums로 정렬. 만/억 그루핑·단위 붙임 규칙은 `../../korean-design-foundation/references/korean-number-date-units.md`(P0-4)를 따른다. | 관례 + P0-4 |

- 5종을 전부 넣을 필요는 없다. 로고 월 + 사례 + 성과 숫자가 기본 3종, 인증마크·언론 보도는 보안·공공 고객 비중이 높을 때 추가한다. (관례)

## 2. B2B CTA 선택 로직

1차 CTA는 **제품 단가 × 셀프서브 성숙도**로 결정한다. 페이지 전체에서 1차 CTA는 1종으로 통일하고 Hero·중간·말미에 반복한다. (관례)

| 제품 단가(연 기준) | 셀프서브 성숙도 | 1차 CTA | 2차 CTA | 다음 단계 |
|---|---|---|---|---|
| 저단가(수십만 원 이하) | 가입 즉시 가치 체험 가능 | **무료 체험** | 요금 보기 | 이메일 가입, 카드 입력 없음 |
| 중단가(수백만~수천만 원) | 온보딩·설정 지원 필요 | **데모 신청** | 무료 체험(제한판) 또는 자료 다운로드 | 리드폼 3~5필드 → 미팅 |
| 고단가(수천만 원 이상, 커스텀) | 셀프서브 불가 | **견적/도입 문의** | 백서·소개서 다운로드 | 리드폼 5~7필드 허용(리드 질 우선) |
| 단가 높지만 셀프서브 가능 | 체험판 존재 | 무료 체험 | 데모 신청 | 체험 → 영업 접점 전환 |

- 근거: 관례 — 저단가는 영업 개입 비용이 회수되지 않고, 고단가·커스텀은 가격 비공개 + 영업 주도가 통용된다.
- "무료 체험"과 "데모 신청"을 같은 위계로 나란히 두지 않는다 — 선택이 갈리면 둘 다 안 누른다. 2차 CTA는 outline/ghost로 위계를 낮춘다. (관례)
- CTA 문구는 동사형 명령("무료로 시작하기", "데모 신청하기")으로 쓰고, 페이지 안에서 표기를 바꾸지 않는다. (관례)
- **목적지 없는 CTA 금지**: 가입·체험 플로우가 아직 없으면 1차 CTA도 리드폼/문의 섹션으로 연결한다 — 눌렀는데 갈 곳 없는 버튼은 막다른 길(H3 위반). (파일럿 확정, F6)

## 3. 리드폼 길이 원칙

| 원칙 | 내용 | 근거 |
|---|---|---|
| 기본 필드 수 | **3~5개** (이름·회사 이메일·연락처·회사명·문의 내용). 7개 초과 금지. | HubSpot 폼 벤치마크 — 필드가 늘수록 전환율이 떨어지고 3개 안팎이 정점 |
| 트레이드오프 | 필드를 줄이면 전환 수↑ 리드 질↓. 고단가·영업 주도 제품은 필드를 늘려 걸러도 된다 — 리드 질이 수보다 중요할 때. | HubSpot — B2B에서 필드 추가가 리드 질을 높인 사례 보고 |
| 긴 폼의 대안 | 2단계 분할(이메일 먼저 → 상세 나중), 선택 필드에 "(선택)" 명시. 필수가 아닌 필드를 필수로 두지 않는다. | Baymard — 불필요 필드와 긴 폼이 이탈 주요인, 단계 분할이 체감 부담을 낮춘다 |
| 한국 필드 관례 | 연락처는 휴대폰 010 3-4-4 자동 하이픈, 회사 이메일 요구 시 안내 문구 명시. 입력 마스크·검증 타이밍은 `../../korean-design-foundation/references/korean-forms.md`(P0-2) 소관. | 관례 + P0-2 |

## 4. 요금제 표기 관례

| 항목 | 규칙 | 근거 |
|---|---|---|
| 월/연 토글 | 기본 선택은 **연 결제**. 할인은 "연 결제 시 20% 할인"으로 명시하고, 연 요금은 월 환산가 병기("월 24,000원 · 연 결제 기준"). | 관례 — 국내 SaaS 요금 페이지 공통 패턴 |
| 부가세 | **"부가세(VAT) 별도"를 가격 인접(요금 카드 하단)에 고정 표기.** 표기 없이 결제 단계에서 10%가 붙으면 신뢰가 깨진다. | 전자상거래법 총액 표시 취지 + 관례(B2B는 별도 표기 통용) |
| 문의 tier | 최고 tier는 가격 자리에 "문의" + CTA "도입 문의". 가격 칸만 비우지 말고 포함 기능 목록은 다른 tier와 같은 형식으로 채운다. | 관례 |
| 숫자 표기 | 원화는 "원" 또는 ₩, 천 단위 콤마, tabular-nums 정렬. 만/억 축약 규칙은 `korean-number-date-units.md`(P0-4). | P0-4 |
| tier 수 | 3~4개. 추천 tier 1개에만 "인기"/"추천" 배지. | 관례 — 5개 이상은 비교 자체가 과업이 된다 |

## 5. 한국형 랜딩 섹션 순서

| 순서 | 섹션 | 내용 | 비고 |
|---|---|---|---|
| 1 | Hero | 헤드라인 + 서브카피 + 1차 CTA | 수치는 `landing.md` |
| 2 | 사회적 증거 | 고객사 로고 월(+성과 숫자 한 줄) | Hero 직후 고정 — 하단으로 미루지 않는다 |
| 3 | 기능/가치 | 문제 → 해결 구조, 특징 2~3열 | |
| 4 | 도입 사례 | 후기·사례 카드(실명·직함·수치) | |
| 5 | 요금/CTA | 요금표(또는 문의) + 1차 CTA 반복 | 말미 CTA는 Hero와 같은 문구 |

- 근거: 관례 — 국내외 B2B SaaS 공통 골격. 사회적 증거가 Hero 직후인 이유는 스크롤 이탈 전 첫 화면 안에서 신뢰 판단이 끝나기 때문.
- 보조 섹션(FAQ·언론 보도·인증마크)은 4~5 사이 또는 푸터 위에 배치한다. (관례)

## 6. 리드폼 인라인 동의 자리

| 항목 | 규칙 |
|---|---|
| 자리 | **제출 버튼 바로 위.** 체크박스 + 한 줄 라벨("개인정보 수집·이용에 동의합니다") + [보기] 링크 |
| 분리 | 필수(수집·이용)와 선택(마케팅 수신) 체크박스 분리, 선택 항목에 "(선택)" 표기 |
| 금지 | 사전 체크(pre-checked) 금지. 필수 동의 없이 제출 버튼 활성화 금지 |
| 터치 | 체크박스+라벨 전체를 클릭 영역으로, `min-h-[44px]` 준수 |

- 근거: 개인정보 보호법 제22조(동의를 받는 방법) + 관례.
- 이 파일은 **자리와 형태만** 정한다. 고지 문구·법정 항목·전체 동의·만 14세 처리는 P1 `korean-consent-legal-ui.md`(foundation/references, 예정) 소관.
